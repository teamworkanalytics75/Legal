Master Plan — Usage Guide
=========================

Purpose
-------
- Central, source‑of‑truth plan for all agents/contributors.
- Machine‑readable JSON + human‑friendly Markdown.

Files
-----
- MASTER_PLAN.json — authoritative task database.
- MASTER_PLAN.md — curated, scannable overview and runbook.

Workflow
--------
- Read MASTER_PLAN.md first to get oriented.
- Update tasks in MASTER_PLAN.json (status, priority, notes).
- Keep MASTER_PLAN.md aligned (summary only; JSON is the truth).

Conventions
-----------
- Status: pending | in_progress | blocked | done | dropped
- Priority: P0 (now), P1 (next), P2 (later)
- Areas: data | modeling | search | workflow | mlops | docs
